## 🚨 BLOCKING — Must pass before commit

### 1. Mobile responsive (320px is the floor)

```bash
# Quick check: open Chrome DevTools → 320x568 viewport
# OR use Playwright:
node -e "
const {chromium} = require('playwright');
(async () => {
  const b = await chromium.launch();
  const ctx = await b.newContext({viewport: {width: 320, height: 568}});
  const p = await ctx.newPage();
  await p.goto('http://localhost:3000/<page>?nocache=' + Date.now());
  const overflow = await p.evaluate(() => document.body.scrollWidth - window.innerWidth);
  console.log(overflow > 0 ? '❌ OVERFLOW ' + overflow + 'px' : '✅ OK');
  await b.close();
})();
"
```

- [ ] **No horizontal scroll on 320px** — `body.scrollWidth === window.innerWidth`
- [ ] **Sidebars collapse on mobile** — `<aside hidden lg:block>` + `<details lg:hidden>`
- [ ] **Buttons collapse to icons** — text inside `<span hidden sm:inline>` + icon `sm:hidden`
- [ ] **Touch targets ≥ 44px** — Apple HIG minimum for tap targets
- [ ] **Text doesn't overflow containers** — `line-clamp-2` or `truncate`

### 2. Cards in grids (the classic "min-w-0" trap)

When rendering cards inside a `<Link>` or grid:

```tsx
// ❌ BAD — cards will overflow on mobile
<a className="block ...">
  <h3>{title}</h3>
</a>

// ✅ GOOD — allows shrinking
<a className="block ... min-w-0 max-w-full w-full">
  <h3>{title}</h3>
</a>

// And the parent grid:
<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-5 min-w-0">
```

- [ ] Every card in a grid has `min-w-0 max-w-full`
- [ ] Grid container has explicit `grid-cols-1` (not relying on default)
- [ ] Grid container has `min-w-0` (otherwise children force intrinsic min-width)
- [ ] Long text is clipped (`line-clamp-N`) not overflowing

### 3. Buttons inside <Link> (event propagation)

When a card is a `<Link>` and contains a button:

```tsx
<Link href="...">
  <button onClick={handleClick}>
    {/* Always stop propagation */}
  </button>
</Link>

function handleClick(e) {
  e.preventDefault();
  e.stopPropagation();  // ← CRITICAL: prevents Link navigation
  // ... do action
}
```

- [ ] Every interactive element inside `<Link>` calls `e.stopPropagation()`
- [ ] Buttons that toggle state use `aria-pressed`
- [ ] Icon-only buttons have `aria-label` + `title`

### 4. Redirect URLs match actual routes

```bash
# Quick check: every `window.location.href` or `router.push` value must exist
grep -rn "router.push\|window.location" src/ | grep -v node_modules
# Then for each URL, verify it exists:
curl -I https://examanet.com/<url>
```

Common mistakes:
- ❌ `/login` (404) → ✅ `/connexion`
- ❌ `/signup` → ✅ `/inscription`
- ❌ `/profile` → ✅ `/mon-compte`

- [ ] All redirect targets return HTTP 200 on prod

### 5. Empty states & fallbacks

```tsx
// ❌ BAD — leaves empty space
{resource.summary && <p>{resource.summary}</p>}

// ✅ GOOD — always renders something
{resource.summary ? (
  <p>{resource.summary}</p>
) : (
  <p className="text-slate-400 italic">Pas de résumé disponible.</p>
)}
```

- [ ] Every optional field has a fallback (italic gray text)
- [ ] Counters = 0 still display (grayed out, not hidden)
- [ ] Lists have an "empty state" UI ("Aucun résultat", "Pas encore d'avis")
- [ ] Images have alt text or fallback icon

### 6. i18n / RTL support

```tsx
import { isArabic } from '@/lib/text-utils';

// Auto-detect RTL based on content
const isAr = isArabic(title);
<h3 dir={isAr ? 'rtl' : 'ltr'} lang={isAr ? 'ar' : 'fr'}>
  {title}
</h3>
```

- [ ] Arabic titles/summaries have `dir="rtl"` + `lang="ar"`
- [ ] French text has `dir="ltr"` + `lang="fr"` (or omit)
- [ ] Mixed-language cards handle both directions correctly
- [ ] Test with cookie `locale=ar` set

## ⚠️ SHOULD — Nice to have

### 7. Accessibility (a11y)

- [ ] All icon-only buttons have `aria-label`
- [ ] All toggleable buttons have `aria-pressed`
- [ ] Forms have `<label>` for every input
- [ ] Color contrast ≥ 4.5:1 for body text, ≥ 3:1 for large text
- [ ] Keyboard navigation works (Tab + Enter)

### 8. Performance

- [ ] No layout shift on load (CLS < 0.1)
- [ ] Images have width/height or aspect-ratio set
- [ ] Long lists are paginated (not all 3829 cards rendered at once)
- [ ] No console errors / warnings in production build

### 9. Optimistic UI

For interactive elements (favorites, likes, ratings):

```tsx
const [favorited, setFavorited] = useState(initial);
const [loading, setLoading] = useState(false);

async function handleClick() {
  if (loading) return;  // prevent double-click
  setLoading(true);
  
  // Optimistic update
  const prev = favorited;
  setFavorited(!prev);
  
  try {
    const res = await fetch('/api/favorites/...', { method: 'POST' });
    if (!res.ok) throw new Error();
    const data = await res.json();
    setFavorited(data.favorited);  // sync with server
  } catch {
    setFavorited(prev);  // revert on error
    toast.error('Erreur — réessayez');
  } finally {
    setLoading(false);
  }
}
```

- [ ] Loading state during async actions
- [ ] Optimistic update with revert on error
- [ ] Toast feedback (success + error)
- [ ] Disabled state while loading

## 🔐 SECURITY — Always check

### 10. No hardcoded credentials

```bash
# BEFORE EVERY COMMIT:
git diff --cached | grep -E "postgresql://.*:[a-zA-Z]+@|BLOB_TOKEN *= *[\"'][a-zA-Z]|API_KEY *= *[\"'][a-zA-Z]"
# Should return NOTHING.
```

- [ ] All secrets via `process.env.X` or `os.environ['X']`
- [ ] No inline `postgresql://user:password@host/db` strings
- [ ] No hardcoded API keys or tokens
- [ ] `.gitignore` covers `extract-*.cjs`, `import-*.cjs`, `migrate-*.cjs`, `cleanup-*.cjs`, `*-credentials.*`

## 🎓 Lessons from past incidents

### 2026-07-02 — Mobile overflow
**What broke** : 110px horizontal scroll on 320px because filter sidebar always visible
**Lesson** : Sidebars need `hidden lg:block` + mobile collapsible fallback

### 2026-07-02 — Cards overflow 6px
**What broke** : Cards forced to 326px in 320px viewport
**Lesson** : Always `min-w-0 max-w-full` on grid children, `min-w-0` on grid container

### 2026-07-02 — Favorite redirect /login (404)
**What broke** : `window.location.href = '/login'` but site uses /connexion
**Lesson** : Always verify redirect target returns 200

### 2026-07-02 — Summary extraction leaked Neon credentials
**What broke** : One-off script with inline DATABASE_URL committed to git
**Lesson** : ALWAYS use `process.env.DATABASE_URL`, never inline credentials

### 2026-06-24 — __pycache__/.pyc with leaked credentials
**What broke** : Python bytecode contained hardcoded Neon password
**Lesson** : `.gitignore` must cover `__pycache__/`, `*.pyc`, etc. before any .py file
